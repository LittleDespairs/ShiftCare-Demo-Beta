Beta Checklist

Рабочий чек-лист для бета-цикла. Линейка версий теперь фиксируется как 0.12.x_beta.

Политика версий

- Базовая бета-линейка: 0.12
- Первая бета-сборка: 0.12.1_beta
- Следующие сборки: 0.12.2_beta, 0.12.3_beta и далее
- Не переходить на 0.13.x и другую minor-ветку, пока цели бета-этапа не закрыты

Цели бета-этапа

- Стабилизировать текущий сценарий работы с расписанием для реального использования

- Закрыть самые болезненные UX- и data-safety-проблемы до перехода к release candidate

Примечания

- Этот файл специально шире, чем финальный список must-fix задач
- По мере поступления beta feedback пункты нужно перераспределить по категориям critical, important и nice-to-have
- Проверено в текущем workspace: backend-тесты проходят в `.venv`, приложение импортируется с `APP_VERSION = 0.12.1_beta`, основные страницы отдают `200` через `TestClient`
- Дополнительно проверено: инициализация чистой SQLite-базы работает во временной директории, packaged-mode логика копирует bundled DB в runtime-директорию, невалидные API-запросы возвращают `422`
- Тестовое покрытие расширено API-регрессиями для CRUD сотрудников, CRUD должностей, CRUD шаблонов смен, upsert/delete недельных пожеланий и сценариев статуса/очистки расписания
- Тестовое покрытие теперь также включает сохранение app settings, сохранение весов генерации и end-to-end недельный сценарий автогенерации для двух позиций
- Экран расписания теперь содержит отдельную post-generation summary panel с метриками прогона, блоками hard/soft constraints, оставшимися незакрытыми требованиями и деталями warning'ов вместо одного длинного сообщения
- Сводка генерации и связанные уведомления локализованы для всех поддерживаемых языков, включая заголовки summary, счётчики, поясняющие заметки, статусы выполнимости и backend-причины из предупреждений
- Рутинные действия на экране расписания стали тише: загрузка, правки смен и смена статусов больше не спамят лишними success-уведомлениями, при этом ошибки и важные итоги генерации остаются видимыми
- Даты с проблемами генерации теперь подсвечиваются прямо в заголовках таблицы и в строке coverage, поэтому от summary-диагностики проще перейти к конкретным дням, которые нужно править вручную
- Валидация Excel-экспорта теперь покрывает пустые недели и смешанные случаи с обычной сменой, `no_show` и `day_off` через проверки содержимого workbook
- Из оставшихся шаблонов убраны legacy-строки `0.11.3_alpha` в asset-version и footer version, поэтому runtime-шаблоны теперь последовательно указывают на beta-версию
- Регрессионное покрытие теперь включает сценарий ручного удаления смены после автогенерации с корректным восстановлением `day_off`; подпункт про сотрудников только на ночные остаётся открытым, потому что такой capability отдельно не смоделирован в текущем приложении
- Для destructive actions теперь подгружается impact preview перед подтверждением на сотрудниках, должностях, назначениях, шаблонах смен и очистке недели, поэтому пользователь видит количество связанных записей до удаления
- Автотесты теперь покрывают delete-impact preview endpoints и проверяют, что `get_base_path()` использует `sys._MEIPASS` в frozen mode
- В приложении теперь есть рабочий backup/restore flow в настройках и автоматические recovery backup перед удалением сущностей и очисткой недели
- Ревизия схемы привела к добавлению дополнительных SQLite-индексов для назначений, недельных пожеланий и дневных статусов, а ревизия validation boundaries добавила более строгие проверки request-models для конфликтных флагов и невалидных временных окон
- Cross-table cleanup теперь покрыт регрессиями для каскадного удаления сотрудника и должности, а удаление шаблона смены намеренно блокируется, пока на него ещё ссылаются записи расписания
- Excel export теперь содержит отдельный coordinator summary sheet, а packaged-mode верифицирован реальной PyInstaller-сборкой: собранный `.exe` отвечал `/schedule` с `200`, используя bundled templates, static assets и DB из `_internal`
- Решение для beta-линейки `0.12`: import tools отложены; поддерживаемый путь data-safety для этой ветки — backup/restore
- Логика построения Excel-export вынесена из `main.py` в [excel_export.py](D:\WebStorm_Projects\PetProjects\Schedule_app\excel_export.py), что уменьшило одну из самых перегруженных зон основного модуля приложения
