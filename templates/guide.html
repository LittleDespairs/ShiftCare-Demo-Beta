User Guide

A simple explanation of how to prepare employees, collect preferences, create a schedule, and export it.

What this app is for

Schedule App helps you build a weekly work schedule. You enter employees, roles, shift times, staffing needs, and personal preferences. Then you can create the schedule automatically or adjust it by hand.

Before creating a schedule

Fill in the basic information once, then update it only when something changes.

  1. Create employees.
  2. Create positions.
  3. Create shift templates.
  4. Assign employees to positions.
  5. Define coverage requirements.
  6. Collect weekly preferences.
  7. Load, generate, review, edit, and export the schedule.
Recommended order: finish setup screens first, then generate the schedule. Most generation problems are caused by missing assignments, inactive shift templates, or coverage requirements that need more staff than are available.

Employees

The Employees page contains the staff list. Use Add employee to open the modal form. Use Edit in the table to open the same modal with the employee data already filled in.

  1. Enter the full name exactly as it should appear in exports.
  2. Choose sex when gender-specific coverage is used.
  3. Set minimum, target, and maximum shifts per week. Target should be between minimum and maximum.
  4. Enable only the work rules that are allowed for that employee, such as nights, weekends, evening after night, or morning plus evening on the same day.

Positions, assignments, and shift templates

Positions describe where people can work. Shift templates describe the real shift times. Assignments connect employees to the positions they can cover.

  1. Create every role that needs its own schedule, such as caregiver, nurse, or coordinator.
  2. Create active shift templates for each position. A template must cover the time interval required by coverage rules.
  3. Assign employees to every position they are allowed to work. The generator ignores employees who are not assigned to the selected position.
  4. Use generation limits on positions only when a role needs stricter rules than the global settings.

Coverage requirements

Coverage requirements tell the app how many people are needed during each part of the day. They are the main input for automatic generation and coverage warnings.

  1. Choose the position and day of week.
  2. Enter start and end times for the required interval.
  3. Set the total required staff and optional female or male minimums.
  4. Keep requirements realistic. If the required number is higher than assigned available employees, the generator will report a shortage.

Collect weekly preferences

Before planning the week, open Preferences and mark days off, vacations, preferred shifts, and restrictions. This helps the system avoid assigning people when they are unavailable.

  1. Select the week and employee.
  2. Mark days off, vacation, sickness, or preferred shifts before running generation.
  3. Clear a preference when it no longer applies. The schedule can then be regenerated or edited manually.
Open preferences

Create the weekly schedule

Open Schedule, choose the week and position, then load the table. Use Auto generate to let the system fill shifts. Review the warnings, then make any manual changes you need.

  1. Choose the first day of the week.
  2. Choose the position you want to schedule.
  3. Click Load to see the current schedule.
  4. Open Generate and choose whether to generate the selected position or all positions.
  5. Check the generation summary, warnings, and coverage rows before you finish.
  6. Use Output to export files, or Danger zone only when you really need to clear schedule data.
Open schedule

Manual changes

You can add or delete shifts manually. You can also mark a person as sick, day off, or no-show. Use these options when the real situation changes after the schedule was created.

  1. Click an empty day cell to choose an active shift template.
  2. Use the status controls to mark day off, sickness, vacation, or no-show when needed.
  3. After manual edits, check coverage again. Manual edits can create new shortages or rule warnings.

Share the schedule

When the week looks correct, open Output on the Schedule page and export the schedule. Excel is useful for editing and review. Word is useful for people who want a printable table without Excel.

  1. Export Excel for the selected position when you only need one schedule.
  2. Export all Excel when you need every position in one workbook.
  3. Export Word or Export all Word when you need a document with visible bordered tables.

Backups and safe destructive actions

The app creates recovery backups before destructive actions such as deleting employees, positions, assignments, templates, or clearing schedules. You can also create and restore manual backups from Settings.

  1. Before deleting, read the modal preview. It shows related records that can be affected.
  2. Use Settings > Data safety to create a manual backup before major changes.
  3. Restore a backup only when you are sure, because it replaces the current database. A pre-restore copy is created first.

If something looks wrong

  1. Check that the employee is assigned to the selected position.
  2. Check that the employee is not blocked by a day off, vacation, sickness, or request.
  3. Check that the shift template is active and has the correct time.
  4. Check Coverage requirements to make sure the needed number of people is correct.
  5. If buttons or modals look stale after an update, refresh the page once so the browser loads the newest scripts.
